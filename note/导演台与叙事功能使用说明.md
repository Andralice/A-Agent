# 导演台与叙事功能使用说明（含 M1～M10）

本文说明 **本仓库叙事与导演台相关能力** 如何配置与上手；**HTTP 契约与字段** 以 **[API接口总表.md](./API接口总表.md)** 为准，**`writingStyleParams` / `narrative`** 见 **[前端与用户指引.md](./前端与用户指引.md)**，里程碑定义见 **[架构1.0.md](./架构1.0.md) §7 / §11 / §12**。

---

## 〇、M1～M10 一览：做什么、你怎么用

| 代号 | 名称（摘要） | 你怎么用 | 关键开关 / 入口 |
|------|----------------|----------|-----------------|
| **M1** | Profile + Prompt 接入 | 选 **`WritingPipeline`**（`POST .../pipeline` 或创建时）；用 **`POST .../writing-style-params`** 写根级 JSON，内嵌 **`narrative`** 对象覆盖情绪/禁止项等；生成后看 **`GET .../generation-tasks`** / 日志 | **`novel.narrative-engine.enabled`**（总闸） |
| **M2** | 叙事 Planner（章前 JSON → 脚本） | `enabled` 且 **`two-phase-enabled: true`** 时章前多一次 Planner；失败会静默回退。轻小说若开 **章拍规划**，默认 **不跑** 叙事 Planner（见 **`two-phase-skip-with-light-novel-plan`**） | **`two-phase-enabled`**、**`planner-temperature`**、**`planner-max-tokens`** |
| **M2′** | 轻小说「章拍规划」（另一路节拍） | **`novel.light-novel.chapter-planning-enabled: true`** 时轻小说章前可先有节拍规划再写正文（与叙事 M2 Planner 不同路，注意别双开浪费） | **`novel.light-novel.chapter-planning-enabled`** |
| **M3** | Lint + 可选窄幅修订 | 终稿后规则扫描；**`lint-fix-enabled: true`** 且命中时多一次窄幅润色 | **`lint-enabled`**、**`lint-fix-enabled`**、**`lint-fix-*`** |
| **M4** | 跨章承接 `narrativeCarryover` | 章成功落库后服务端维护；**无**单独 HTTP 只改承接。读 **`GET /api/novel/{id}`** 的 **`narrativeCarryover`**；初稿自动注入 | **`m4-carryover-enabled`** |
| **M5** | 情绪波形 + 管线句法力学 | 随叙事引擎注入提示词，无需单独 API；调管线 + **`narrative`** 与默认 Profile 即可改变读感 | 依赖 **`enabled`**；逻辑在 `NarrativeCraftPrompts` / `NarrativeProfileDefaults` |
| **M6** | 叙事物理双引擎 | 在 **`writingStyleParams`** 根级设 **`narrativePhysicsMode`**（如 **`continuous`** / **`stress`** 等，见架构 §11）覆盖管线默认 | 根字段 **`narrativePhysicsMode`**；默认由 **`WritingPipeline`** 决定 |
| **M7** | 章节叙事侧车 JSON | 需 **`enabled`** 且 **`m7-artifact-enabled: true`**。只读：**`GET .../narrative-artifacts`**、**`GET .../chapters/{n}/narrative-artifact`**（无数据时 **`artifact` 为 null**） | **`m7-artifact-enabled`** |
| **M8** | 批评 JSON + 可选一轮重写 | **`m8-critic-enabled`** 打开批评；**`m8-rewrite-enabled`** 且达到 **`m8-rewrite-min-severity`** 时窄幅重写。侧车 JSON 里可有 **`critic`** 块 | **`m8-critic-enabled`**、**`m8-rewrite-enabled`**、**`m8-*-temperature`**、**`m8-*-max-tokens`**、**`m8-rewrite-min-severity`** |
| **M9** | 书本跨章快照 `narrative_state_json` | **`m9-crosscut-enabled: true`** 后每章落库更新；**`GET .../narrative-state`**；第 2 章起初稿带 M9 摘录块；含 **v1.1**：**`relationshipHints`**、**`tensionRippleHint`**、**`schemaVersion`** | **`m9-crosscut-enabled`** |
| **M10** | 前端导演台（本仓库静态页） | 浏览器打开 **`/novels.html`**：运行时条、按书导演台（监控 / M9 / pipeline / 任务、多种 POST、cancel/kick/retry）。**产品 UI** 仍在 **web-agent** | **`static/novels.html`**；与 API 同源 |

## 一、`application.yml` 中与 M1～M9 相关的键（速查）

以下均在 **`novel.narrative-engine`** 下，除非另起一行注明。

| 配置键 | 关联 | 说明 |
|--------|------|------|
| **`enabled`** | M1～M8 总闸 | `false` 时叙事引擎大块不注入，M7 侧车也不会写。 |
| **`two-phase-enabled`** | M2 | 叙事 Planner 开关；仓库默认 `true`。 |
| **`two-phase-skip-with-light-novel-plan`** | M2 vs M2′ | 轻小说已走章拍规划时是否跳过叙事 Planner。 |
| **`planner-temperature`** / **`planner-max-tokens`** | M2 | Planner 调用参数。 |
| **`lint-enabled`** / **`lint-fix-enabled`** / **`lint-fix-*`** | M3 | Lint 与命中后窄幅修订。 |
| **`m4-carryover-enabled`** | M4 | 承接字段更新与初稿注入。 |
| **`m7-artifact-enabled`** | M7 | 章节 **`narrative_engine_artifact`** 写入（仍要求 **`enabled: true`**）。 |
| **`m8-critic-enabled`** / **`m8-rewrite-enabled`** / **`m8-*`** | M8 | 批评与重写；仓库默认开，多 1～2 次模型调用；可关以省费用。 |
| **`m9-crosscut-enabled`** | M9 | 书本 **`narrative_state_json`** 与 **`GET .../narrative-state`**、初稿 M9 块；仓库默认开。 |

**轻小说章拍（与叙事 M2 并列的另一开关）**：

```yaml
novel:
  light-novel:
    chapter-planning-enabled: true   # M2′：章前节拍规划，再写正文
```

修改后需 **重启 Spring Boot**；前端可用 **`GET /api/novel/config/frontend-runtime`** 核对部分运行时值（与部署 yml 一致）。

## 二、M1～M10 分条使用说明

### M1：Profile + Prompt

- **作用**：按 **`WritingPipeline`** 与 **`writingStyleParams`**（含 **`narrative`**）合并成有效 Profile，拼进初稿 / 润色 / 终稿提示词。  
- **你怎么做**：创建或续写前设好书 **`POST .../pipeline`**、**`POST .../writing-style-params`**；在 **`/novels.html` 导演台** 里也可改 pipeline / 微参 JSON。  
- **验收**：同题材对比开关 **`enabled`** 前后读感；**`GET .../creative-process`** 或生成日志里可见叙事相关摘要（若已记录）。

### M2：叙事 Planner（两阶段）

- **作用**：章前多一次低温调用，产出 JSON 叙事脚本再写初稿；解析失败则 **静默跳过**（不挡主流程）。  
- **你怎么做**：设 **`two-phase-enabled: true`**；按需调 **`planner-temperature`**、**`planner-max-tokens`**。  
- **与轻小说章拍**：若 **`chapter-planning-enabled: true`**，通常保持 **`two-phase-skip-with-light-novel-plan: true`**，避免两套节拍重复扣费。

### M2′：轻小说章拍规划

- **作用**：**`novel.light-novel.chapter-planning-enabled`** 为 **`true`** 时，轻小说管线章前可先跑「节拍规划」再写正文（与 **M2 叙事 Planner** 不同实现）。  
- **你怎么做**：在 yml 开关；与 **M2** 是否同开见上表 **「〇」**。

### M3：Lint + 窄幅修订

- **作用**：终稿后规则扫描禁止短语/标签句；可选 **`lint-fix-enabled`** 触发一轮窄幅润色。  
- **你怎么做**：调 **`lint-enabled`** / **`lint-fix-enabled`** 及 **`*-temperature`**、**`*-max-tokens`**。

### M4：跨章承接

- **作用**：**`novel.narrativeCarryover`** 在章落库后更新，**下一章初稿**注入承接块。  
- **你怎么做**：保持 **`m4-carryover-enabled: true`**；用 **`GET /api/novel/{id}`** 看 **`narrativeCarryover`**。  
- **注意**：**没有**「只改承接」的 HTTP 接口；改文风微参、重写章节会间接影响后续承接。

### M5：情绪波形与句法力学

- **作用**：提示词内追加 M5 硬约束块（波形、管线句法、互动显微模板等），随管线/Profile 变化。  
- **你怎么做**：无需单独 API；保证 **M1** 配置合理即可；对比不同 **`pipeline`** 与 **`narrative`**。

### M6：叙事物理模式

- **作用**：**连续微扰** vs **压力阈值** 两套动力学提示分桶，避免混用语义。  
- **你怎么做**：在 **`writingStyleParams` JSON 根** 增加 **`"narrativePhysicsMode":"continuous"`** 或 **`"stress"`** 等（完整别名见 **架构1.0 §11** / 代码 **`NarrativePhysicsMode.parseOverride`**）；或通过 **`POST .../pipeline`** 换默认管线（轻小说/日常默认 continuous 系，爽文等默认 stress 系）。

### M7：章节叙事侧车（结构化 JSON）

- **作用**：Planner/Lint 等结构化结果写入 **`Chapter.narrativeEngineArtifact`**，便于排障与对照。  
- **你怎么做**：**`enabled`** + **`m7-artifact-enabled: true`**；调用 **只读**：  
  - **`GET /api/novel/{novelId}/narrative-artifacts`**  
  - **`GET /api/novel/{novelId}/chapters/{chapterNumber}/narrative-artifact`**  
- **验收**：有生成过的章应能拉到 **`artifact`**；无数据为 **`null`** 属正常。

### M8：批评—重写（可选）

- **作用**：Lint 后可跑批评 JSON；达严重度阈值可再跑 **一轮** 窄幅重写（非 N-best 默认）。  
- **你怎么做**：按需 **`m8-critic-enabled: true`**、**`m8-rewrite-enabled: true`**，并调 **`m8-rewrite-min-severity`**（`low` / `medium` / `high`）。注意 **费用与延迟** 上升。  
- **验收**：M7 侧车 JSON 中 **`critic`** 相关字段（若实现已写入）。

### M9：书本跨章快照

- **作用**：**`narrative_state_json`** 聚合 M4 预览、侧车、近期事实、阶段快照；**v1.1** 增加 **`relationshipHints`**、**`tensionRippleHint`**、**`schemaVersion`**；第 2 章起初稿带摘录块。  
- **你怎么做**：**`m9-crosscut-enabled: true`** 并重启；至少成功落库一章后再看 **`GET .../narrative-state`**；**第 1 章**无 M9 注入属预期。  
- **导演台**：弹层内 **「跨章叙事 M9」** 即该 GET 的 JSON。

### M10：导演台（同源页 + 产品前端）

- **本仓库**：**`http://<host>:8080/novels.html`** — 运行时条、按书 **导演台**（监控、M9、pipeline、任务、**pipeline / hot-meme / writing-style-params / book-meta / continue / auto-continue**、**cancel / kick / retry**）。详见下文 **第三节**。  
- **产品前端**：在 **web-agent（Electron）** 中实现同等 API 调用；**`/novels.html` 非产品界面**，仅联调/运维对照。

## 三、同源联调页 `/novels.html`（导演台操作步骤）

### 1. 打开与登录

- 访问 **`http://<host>:8080/novels.html`**（与 API 同源，一般无 CORS）。  
- **`app.security.enabled: true`** 时，访问非公开书需 JWT；页内 **登录** 后 token 存 **localStorage**。

### 2. 页顶「运行时配置」

- 自动 **`GET /api/novel/config/frontend-runtime`**；可核对 **M7 / M9** 等开关是否与服务器 yml 一致。  
- **「刷新运行时配置」** 用于改配置重启后再拉。

### 3. 书列表

- **刷新列表**、**新建**、**书库可见性**、**删除**（delete-guard → delete）与 API 总表一致。

### 4. 「导演台」弹层（按书）

| 区块 | 对应能力 | 说明 |
|------|-----------|------|
| 写作监控 | 任务/锁/章节状态 | `GET .../writing-monitor` |
| 跨章叙事 M9 | M9 | `GET .../narrative-state` |
| 流水线快照 | M1/M6 等上下文 | `GET .../pipeline` |
| 写路径·全书 | M1/M6、热梗、备注 | **pipeline** / **hot-meme** / **writing-style-params** / **book-meta** |
| 生成任务与操作 | 队列 | **generation-tasks** + **cancel / kick / retry** |
| 续写一章 | 创作 | **continue** |
| 自动续写 | 创作 | **auto-continue**（`targetChapterCount` 须大于当前已写章数，上限见 **frontend-runtime**） |

弹层 **「刷新本面板」** 会重拉上述 GET，并用 **pipeline** 回包回填表单（含书籍备注框）。

## 四、产品前端（web-agent）建议

1. 启动或进入创作域：**`GET /api/novel/config/frontend-runtime`**。  
2. 异步任务：轮询 **`GET .../writing-monitor`** 或 **`GET .../tasks/{taskId}`**；需要时 **cancel / kick / retry**。  
3. **`pipeline`** 与 **`novel-style` URL 的 `{style}`**：见总表 **`fromPath`** 别名表；续写实际以 **`GET .../pipeline`** 为准。  
4. **M7 / M9**：按上表只读接口拉取；**M9** 注意首章与 **`null`** 预期。
