# A-Agent 系统功能说明（已实现能力）

本文从**产品与功能**角度归纳当前后端（及同源静态页）**已经做到的事**，便于对齐预期与对外说明。**不写 HTTP 路径与请求体**；契约与字段仍以 **[API接口总表.md](./API接口总表.md)** 为准，叙事里程碑细节见 **[架构1.0.md](./架构1.0.md)** §11 / §12、上手配置见 **[导演台与叙事功能使用说明.md](./导演台与叙事功能使用说明.md)**。

---

## 1. 系统定位

- **单体 Spring Boot 应用**：连接 **MySQL** 持久化小说与任务，通过 **兼容 OpenAI 协议** 的对话模型完成生成。
- **双入口**：**浏览器 / 前端** 走 REST；**QQ 群（NapCat）** 可走 Webhook + 异步派发，与同一套领域服务共用。
- **章节正文交付形态**：以**异步生成任务 + 轮询查询结果**为主；**不提供**「章节正文逐字流式输出」的 HTTP 能力（若产品上需要流式，须另行约定协议与实现）。

---

## 2. 书籍与内容生命周期

- **开书**：按题材与生成设定创建小说；可绑定**文风流水线**、可选**文风微参 JSON**、**连载平台/创作者备注**等书本级信息。
- **大纲**：生成并保存全书大纲正文；默认 **两阶段（效果最好）**：先专用调用产出 **冲突图谱 JSON**（setup / conflicts / turning_points / tension_matrix / possible_branch_hints 等），再生成 Markdown 全文并锚定该 JSON；图谱落库字段 **`outlineGraphJson`**，关闭见 `novel.outline.two-phase-graph-enabled`。另 **按 `WritingPipeline` 切换叙事内核**（`outlineNarrativeKernelBlock`，思路见 `note/note2`）；输出要求「冲突结构概要」、张力矩阵与开篇细纲每章 **关系+/目标+/认知+**；支持规划章参与 **按作者提示重生大纲**（同步、耗时可较长）。
- **角色档案**：开书时在**大纲已定稿后**生成；提示词会附带从大纲 Markdown **摘取的角色相关段落**（世界观/主角/主要/对立/冲突概要）以及 **`outlineGraphJson` 中的 setup/conflicts/tension_matrix**，约束 JSON 中 **`name` 与大纲称谓一致**（摘录逻辑见 `OutlineCharacterExcerpt`）。修补重建时设定正文已含全书大纲，额外只锚定冲突图谱以免重复灌全文。支持后续**修补/重建**（含扩展选项）。
- **章节**：按序号生成正文并落库；支持**续写下一章**、**指定区间重生**（带并发守卫，避免同一区间被多处同时改写）。
- **自动续写**：可按书本配置的目标章节数**批量排队**生成，直到达到上限或失败；与任务系统、进度展示联动。
- **导出**：将已有章节导出为可读文本成品（具体格式与选项以实现为准）。
- **删除**：支持带确认条件的**硬删除**书本及关联数据（与删除守卫、权限策略配合）。

## 3. 文风、流水线与书本级开关

- **多条写作流水线**：例如爽文、轻小说、日常向、年代文、粗俗风等；流水线决定默认叙事语气、润色分支与大量提示词策略。
- **文风微参**：如叙事外放程度、对白占比倾向、幽默程度、年代考究程度等，存于书本 JSON，**仅影响后续生成**。
- **叙事对象 `narrative`**：与微参并列，可覆盖**情绪类型、强度带宽、压抑度、触发事实、节奏/材质/视角意图、禁止表述、轻小说向微情绪维、互动写法、是否强调「去标签句仍可读懂」**等；与流水线默认 Profile **合并**后注入生成链路。
- **叙事物理模式**：可在书本 JSON 根级指定「连续微扰」或「压力阈值」类动力学倾向，与流水线默认正交合并，避免两套语义混用作一章的主骨架。
- **全书「网络热梗」开关**：关闭时提示词走克制路径；开启时在合规前提下允许少量、克制的梗用法（已写章节不因改开关而自动改写）。
- **大纲规划默认值**：服务端可提供「开篇细纲章数 / 路线图末章」的默认与允许区间，供创建表单对齐；单本书也可持久化自己的规划参数。

## 4. 叙事引擎（M1～M9，功能层面）

以下均由 **`application.yml` 中叙事引擎相关开关**控制（具体键名见导演台说明）；关闭总闸时大块叙事增强不会注入。

| 能力块 | 已实现的行为（摘要） |
|--------|----------------------|
| **M1** | 按流水线 + 书本 JSON 合并出有效 **Narrative Profile**，嵌入**初稿、润色、终稿去 AI 味**等相关提示词。 |
| **M2** | 章前可选 **Planner**：多一次模型调用产出结构化叙事脚本（含情绪弧等字段目标），成功则注入初稿；失败则静默退回单阶段。**轻小说**若启用另一条「章拍规划」，可与本 Planner **互斥跳过**，避免双重节拍、双倍费用。 |
| **M2′（轻小说章拍）** | 轻小说管线可在写正文前先做**节拍规划**再写正文（服务端 YAML 开关）。 |
| **M3** | 终稿后进行 **Lint**（禁止短语、可选情绪标签类规则）；命中时可触发**窄幅修订**（多一次调用），以删标签、守纪律为主，不改剧情因果。 |
| **M4** | **跨章承接**：章节成功落库后更新书本级承接摘要，**下一章初稿**自动附带承接块，约束开篇情绪与场面接续（禁止复述原文式抄袭）。 |
| **M5** | **情绪波形硬约束** + **按流水线的句法力学** +（条件满足时）**互动显微模板**；润色/终稿阶段附带简短 M5 提醒，强化节奏与去标签。 |
| **M6** | **叙事物理双引擎**提示分桶（连续微扰 vs 压力阈值），与 M5 并用，约束本章主导动力学语义。 |
| **M7** | 将 Planner / Lint / 批评等**结构化中间结果**写入章节侧车字段，便于排查与对照（需总闸与侧车开关同时开启）。 |
| **M8（v1）** | Lint 之后可选 **批评 JSON**；达到配置的严重度阈值时可再跑**一轮窄幅重写**（非 N-best、非无限多轮）。 |
| **M9（v1 + v1.1）** | 章节成功落库后聚合**书本级跨章叙事状态 JSON**（承接预览、侧车摘要、近期事实摘录、阶段快照摘录等），并含 **relationshipHints、tensionRippleHint、schemaVersion** 等深化字段；**从第 2 章起**初稿可附带上一快照的摘录块。**不含**独立「关系图边表」或「可学习余波模型」——仍为后续愿景。 |

## 5. 连续性、事实与侧车（主线写作辅助）

- **章节事实、叙事记忆、阶段主线快照**：用于支撑跨章设定一致性与大纲对齐（具体抽取与写入策略在领域服务内实现）。
- **章节侧车（Sidecar）**：保存与本章生成相关的结构化锚点、事实摘要等，与 M4/M9 及事实表同源摘要，避免多套互不通的「平行记忆」。
- **实体一致性检测与告警**：可记录一致性方面的告警信息，供运维或后续产品展示（不等同于自动改写全书）。

---

## 6. 任务、进度与可观测

- **异步任务表**：章节生成、大纲重生租约等以任务状态机持久化；支持超时失败、取消、重试、Kick 等运维动作（与并发守卫配合）。
- **写作进度**：持久化当前写作阶段、区间、游标等，供监控页或前端聚合展示。
- **创作时间线**：可从服务端拉取本书关键事件序列（创建、大纲、章节里程碑等），便于产品做「进程叙事」展示。
- **生成日志**：记录生成动作的结果摘要；叙事引擎开启时可将叙事相关摘要写入日志备注字段，便于检索。
- **可选 AI HTTP 请求日志**：调试用途，默认关闭；开启时应注意日志体量与敏感信息。

---

## 7. 安全与书库可见性

- 可启用 **JWT + 管理员账号**：登录后获得管理员权限。
- **书本级可见性**：可标记为公共或对普通用户隐藏（仅管理员可见）；关闭安全总闸时行为可与旧版一致（全部可见、无登录），由部署配置决定。

---

## 8. QQ / NapCat 侧能力

- 接收群消息回调，**解析指令**后异步调用与 HTTP 同源的小说编排逻辑（开书、续写、列表、读章、自动续写等，以实现与指令表为准）。
- 可将生成结果**格式化回贴**到群（具体模板与长度策略以实现为准）。
- 连接 NapCat 的 **WebSocket / Token** 等由配置文件提供；群白名单等用于限制调用来源。

---

## 9. 同源静态「导演台」页（工程联调用）

- 浏览器访问应用宿主上的 **`/novels.html`**：提供运行时信息条、按书的导演操作区（如流水线、热梗、文风 JSON、书本备注、续写/自动续写、任务列表与取消/重试等）。
- **定位**：便于运维与前后端**同源联调**，**不是**完整产品 UI；完整结构化表单与编辑室仍在独立前端（如 Electron web-agent）规划。

---

## 10. 实验性 / 相对独立的模块

- **`engine` 包（世界模拟、角色情绪状态等）**：更偏实验或扩展方向，与当前主线「落库写长篇」可并行演进，不应默认等同于生产必开能力。

---

## 11. 当前刻意未实现或仅保留为愿景的能力（避免误判）

- **章节正文 SSE 或等价实时流式输出**（与现有 REST 能力边界分离）。
- **M8 的 N-best / 多轮选择闭环**（现为可选的一轮批评 + 一轮重写）。
- **M9 之后的关系图边表、可学习余波**等深化存储与推理。
- **note1 / 架构 §12 中的演化层、版本轨、多轨成品选择**等产品形态。
- **M11 同人 Canon 层**：远期独立里程碑。
