# A-Agent 目标系统架构 1.0（叙事引擎）

本文描述 **目标态** 架构：在现有单体后端之上，把「文风 / 情绪 / 节奏 / 信息控制」从一次性指令升级为 **可编程、可验收、可演进** 的叙事流水线。  
**当前已实现的后端分层与模块清单**见 [架构.md](./架构.md)；本文与之的关系是：**架构.md = 现状**，**架构1.0.md = 目标与迭代路线**。

---

## 1. 文档定位与范围

| 项 | 说明 |
|----|------|
| 版本 | 1.0 — 冻结「方向与分层」，实施按里程碑增量交付 |

| note1 全文愿景 | 见 **§12**，分期落入 M5～M11；不在单里程碑一次交付 |

## 2. 愿景：从「生成文本」到「叙事状态机」

**目标系统**在接口上仍是一次或多次 LLM 调用，在结构上明确三层：

1. **物理层（约束）**：区间、禁止项、节奏与语言材质规则 —— 保证下限、防止漂移与风格污染。  
2. **智能层（模型）**：在约束内完成表达 —— 提供人物个性与意外感。  
3. **演化层（可选、后续）**：日志化中间产物、抽检、局部重写、跨章状态合并 —— 越用越稳。

一句话：**「约束中的自由」** — AI 可以判断怎么演，但不能独裁路线；规则提供路基，模型负责在车辙内开车。

---

## 3. 设计原则

- **情绪是变量 + 区间，不是单一标签**：用 `intensity_range`、`suppression`、`trigger_fact`（事实句）等限制强度与成因，避免「平均人类反应」替代角色个性。  
- **事件驱动优于口头驱动**：优先写清「发生了什么」，再映射到情绪区间，而非「写愤怒一点」。  
- **风格 = 情绪动力学 + 语言力学**：节奏（句长/断裂）、视角、信息释放、材质（软硬抽象比）与情绪并列配置，避免只调音量不调编曲。  
- **管线分化，事实层共享**：不同 `WritingPipeline` 使用不同默认 **Narrative Profile** 与禁止项；角色、章节事实、快照等 **共享上下文**，避免两套完全割裂的「引擎」重复维护（除非明确 fork 实验管线）。  
- **中间产物显性化**：Planner 输出、应用的 profile、Lint 结果应可写入日志或侧车字段，支持 debug 与 A/B。  
- **失败可降级**：Planner 失败、JSON 解析失败时 **回退单阶段生成**，不阻断主路径。

## 5. 核心概念与数据契约（目标）

### 5.1 Narrative Profile（叙事配置）

最小可用字段建议（可扩展，1.0 实施时可删减）：

| 字段 | 含义 |
|------|------|
| `emotion_type` | 主情绪类型（枚举或短字符串） |
| `intensity_min` / `intensity_max` | 强度区间 |
| `suppression` | 压抑程度 0～1 |
| `trigger_fact` | 触发事实（一句，不含评价） |
| `forbidden_phrases` / `forbidden_patterns` | 禁止项（风格/标签句） |
| `rhythm_hint` | 节奏意图（如：短句主导 / 流动长句 / 断裂停顿） |
| `texture_hint` | 语言材质（软/硬、感官密度倾向） |
| `pov_hint` | 视角远近（可选） |

**轻小说向增量**（可与爽文管线并存为不同默认值）：`affection`、`awkwardness`、`assertiveness` 等「微情绪」标量，强调 **互动模板**（动作 + 对话 + 微反应），避免高频「火山爆发」。

### 5.2 Planner 输出（两阶段，可选）

JSON 形态目标（示例级，非最终实现 Schema）：

- `beats`：有序列表（每 beat：意图 + 建议情绪落地强度或区间子段）。  
- `constraints_echo`：对禁止项与节奏的确认。  
- `character_moment`：本段角色偏差提示（可选）。

### 5.3 Lint / Polish

- **Lint**：规则优先（短语黑名单、句长统计、可选情绪标签检测），输出 issue 列表。  
- **Polish**：窄幅改写 — **不改剧情与对话语义**，主要删标签、调节奏；设 **次数/token 上限**。

### 5.4 跨章状态（1.0 后期）

- 独立 JSON 或与 `Novel` / `Chapter` 扩展字段：`relationship_snapshot`、`emotional_afterglow`、`open_threads` 等。  
- 与现有 **chapter_fact / plot_snapshot** 对齐，避免多套互不通的记忆源。

## 8. 横切能力（沿用现状，目标架构不推翻）

- **任务与互斥**：`GenerationTaskService`、`RegenerationTaskGuardService`、大纲租约等 —— 叙事引擎 1.0 不改变并发契约。  
- **安全与书库**：`NovelLibraryAccessService`、`SecurityConfig` —— 不变。  
- **一致性 / 事实 / 快照**：继续作为 **Writer 上下文输入**；1.0 优先 **读** 而非重写整套一致性架构。

## 9. 风险与约束

- **成本与延迟**：两阶段 + Polish 增加调用次数，必须默认关闭或按管线/按章节类型开启。  
- **过度工程**：禁止在未验证 M1 前并行建设完整 DSL UI + 多引擎状态机。  
- **评测**：自动化 Lint 无法替代抽样人工；保留「删情绪标签读后测」作为廉价金标准。

## 11. 实现状态（后端落地）

- **已入库**：`com.start.agent.narrative`（`NarrativeProfile`、`NarrativeProfileDefaults`、`NarrativeProfileResolver`）；章节链路在 `NovelGenerationAgent` 初稿 / 润色 / 终稿去 AI 味中注入叙事块；`NovelAgentService` 按书合并 `writingStyleParams` 根下 **`narrative`** 对象与管线默认。
- **M2**：`NovelGenerationAgent` 在 **`novel.narrative-engine.two-phase-enabled: true`** 且叙事引擎开启、存在 `NarrativeProfile` 时，章前增加 **Planner** 调用（低温、`planner-max-tokens` 限制），产出 JSON → 格式化为「本章叙事脚本」注入初稿；**失败或 JSON 无效则静默跳过**。轻小说且启用章拍规划时默认 **不跑** 叙事 Planner（`two-phase-skip-with-light-novel-plan`，避免节拍重复）。
- **M3**：终稿去 AI 味之后，`NarrativeLint` 扫描正文：`forbidden` 短语子串命中 +（`readerInferenceRule=true` 时）少量直白情绪标签正则；**`lint-fix-enabled: true`** 时命中则 **`PolishingAgent.polishNarrativeLintPass`** 窄幅修订一轮（低温、`lint-fix-max-tokens`），失败或空结果保留原文；**仅叙事引擎开启且有 Profile 时运行**。
- **M4**：`NovelAgentService.genInner` 在 **`m4-carryover-enabled: true`** 时将 **`novel.narrativeCarryover`** 传入 **`NovelGenerationAgent`** 初稿提示词；**`saveChapter` 成功后**用 **`ChapterSidecar.continuityAnchor`** + 本章正文**尾声摘录**写回书本字段（上限约 1800 字）；与事实表 `continuity_anchor` 同源摘要，供跨章读感连续。
- **M5（情绪波形硬约束）**：`NarrativeCraftPrompts.narrativeEngineM5ChapterHardBlock` — 初稿在叙事引擎块后追加 **波形纪律** + **按 `WritingPipeline` 分桶的句法力学** +（轻小说管线或用户配置了微情绪维/互动写法时）**互动显微模板**；润色与终稿分别通过 `narrativeEnginePolishReminder` / `narrativeEngineDeAiReminder` 追加 **M5 单行约束**。默认 Profile 禁止项在 **`NarrativeProfileDefaults`** 中补强（避免情绪档位贴死与纯旁白交代关系）。
- **M6（叙事物理双引擎）**：`NarrativePhysicsMode`（`CONTINUOUS_MICRO` / `STRESS_THRESHOLD`）由 **`NarrativeProfileResolver.resolvePhysicsMode`** 解析：默认 **轻小说 / 日常 → 连续微扰**，**爽文 / 年代 / 粗俗 → 压力阈值**；书本 JSON 根 **`narrativePhysicsMode`** 可覆盖。初稿注入 **`narrativeEngineM6PhysicsBlock`**，润色/终稿附带 **M6 单行**，避免两套动力学语义混用作主导骨架。
- **配置**：`novel.narrative-engine.*`，见 `application.yml`。
- **M8（批评—重写，可选）**：`NovelGenerationAgent` 在 **M3 Lint** 之后、`m8-critic-enabled: true` 且叙事引擎 + Profile 生效时多一次 **批评 JSON**（`PolishingAgent.callNarrativeCritic`）；`m8-rewrite-enabled: true` 且命中项达到 **`m8-rewrite-min-severity`** 时再 **`polishNarrativeCriticPass`** 一轮窄幅重写。侧车 JSON（M7）含 **`critic`** 块（`status` / `issues` / `rewriteApplied` 等）。仓库 **`application.yml` 默认开启** 批评与重写；部署可改回 `false` 以省调用。
- **M9（跨章叙事状态 v1）**：`novel.narrative-engine.m9-crosscut-enabled: true` 时，`NovelAgentService.genInner` 在每章成功落库后聚合 **M4 承接预览、本章 sidecar、近期 `sidecar_fact`、最新 `plot_snapshot`** 写入 **`novel.narrative_state_json`**；**`GET /api/novel/{novelId}/narrative-state`** 只读解析；**下一章初稿**上下文追加 **M9 快照摘录块**（与事实/快照同源，禁止复述原文）。仓库 **默认开启**；与 `chapter_fact` / `plot_snapshot` 表同源，不另建平行事实源。
- **可观测**：成功章节日志写入 `generation_log.context_note`（叙事摘要，兼容 Hibernate `ddl-auto` 增列）。
- **可选覆盖 JSON**（挂在本书 `writingStyleParams` 内）示例：

```json
{
  "narrativePhysicsMode": "continuous",
  "narrative": {
    "emotionType": "克制对峙",
    "intensityMin": 0.5,
    "intensityMax": 0.85,
    "suppression": 0.45,
    "triggerFact": "证据当面摊开但仍欠一环",
    "forbidden": ["他很愤怒", "一股前所未有的"],
    "rhythmHint": "前紧后松，结尾留一击未落槌",
    "readerInferenceRule": true
  }
}
```

## 12. note1 愿景对齐与 2.0 落地路线图（目标）

[note1](./note1)（仓库内文件 **`note/note1`**，无 `.md` 扩展名；克隆完整 A-Agent 后该路径应存在）描述的是 **「可编程叙事 / 多物理引擎 + 演化层」**：情绪不是标签而是 **变量+区间+触发**；文体差异落到 **节奏、视角、信息释放、语言材质、句法**；轻小说与爽文/粗粝风 **不得共用同一套情绪骨架**；系统应 **显性化中间产物**、可选 **批评—重写—选择** 闭环，并最终走向 **前端调度台（Vue）+ 流式 + 版本** 等产品形态。

**与当前仓库的关系**：§11 的 **M1～M4** 已覆盖 note1 中的 **冰山一角**（Profile、可选 Planner、Lint、跨章承接）。要以工程方式「实现 note1」，需在 **不推翻现有编排** 的前提下，按阶段增量挂载 **情绪动力学模块、引擎路由、持久化侧车、评估闭环、前端协议**。下列里程碑建议 **顺序交付**，每一阶段都可单独验收；并行只做 **前端对接**（见 [API接口总表](./API接口总表.md)）与后端一条竖切线，避免 §9 所述过度工程。

|------|----------------|-----------------|------------------|
