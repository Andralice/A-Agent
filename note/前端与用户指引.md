## 一、书籍备注（连载平台 & 创作说明）

这两项**只给你和读者做展示/项目管理**，不会当作 AI 提示词拼进模型。

| 字段 | 含义 | 典型示例 |
|------|------|----------|
| `serializationPlatform` | 在哪个平台连载 | `起点`、`晋江`、`番茄`、`仅备份未发布` |
| `creatorNote` | 这本书是干什么的、给谁看、备注 | `练习长篇节奏，不入V`、`帮朋友代笔草稿` |

**创建时带上（可选）**

- `POST /api/novel/create`  
  Body 增加：`serializationPlatform`、`creatorNote`（与 `topic`、`generationSetting` 同级）。
- `POST /api/novel-style/{style}/create`  
  同上。

**单独修改（不必重建书）**

- `POST /api/novel/{novelId}/book-meta`  
  Body 示例：

```json
{
  "serializationPlatform": "晋江",
  "creatorNote": "练笔，慢热日常向，不入V"
}
```

只改其中一个字段时，另一个字段**不要出现在 JSON 里**（或保持不传），否则不会误清空。要清空某字段可传空字符串 `""`。

**前端读取**

- `GET /api/novel/{novelId}`：实体里直接有 `serializationPlatform`、`creatorNote`。
- `GET /api/novel/{novelId}/outline`：已包含同上字段。
- `GET /api/novel/{novelId}/pipeline`：同上（便于详情页一次拉齐）。
- `GET /api/novel/{novelId}/creative-process`：时间线第一步 `novel_created` 里也有这两项。

> 注意：`description` / `outline` 仍是 **AI 生成的大纲正文**，与 `creatorNote` 不同。

**大纲规划默认值（创建表单）**

- `GET /api/novel/config/outline-plan-defaults`：拉默认「开篇细纲章数」「路线图末章」及允许区间。

**重新生成大纲（覆盖 AI 大纲正文，可写建议）**

- `POST /api/novel/{novelId}/outline/regenerate`，Body 可选 `hint`（本次希望模型遵守的写作/节奏建议）、以及可选的 `outlineDetailedPrefixChapters` / `outlineMinRoadmapChapters`。
- 同步接口、耗时可能较长；已写章节不会自动改，详见 **[API接口总表.md](./API接口总表.md)「### 4.1」**。

## 二、文风微参 `writingStyleParams`（可选 JSON）

存库字段：`writingStyleParams`（字符串形式的 JSON）。创建或 `POST /api/novel/{novelId}/writing-style-params` 更新；**只影响后续生成**。服务端对合法 JSON **整树写回**（叙事 / 认知弧线 / 文笔四层等与下文四类枚举可并存，勿只提交局部 Tab）。

> **权威规格**：字段 **类型、取值范围、snake_case 别名、校验规则、前端合并建议** 以 **[API接口总表.md](./API接口总表.md) §16.2.1～16.2.6** 为准；**参数并存、M2/轻小说互斥、叙事引擎关闭后的生效范围、语义重叠与优先级** 以 **§16.2.8** 为准。本节为精简备忘。

支持字段（均为字符串，英文枚举，大小写不敏感）：

| 字段 | 可选值 | 含义 |
|------|--------|------|
| `styleIntensity` | `mild` / `balanced` / `bold` | 叙事外放程度 |
| `dialogueRatioHint` | `low` / `medium` / `high` | 对白占比倾向 |
| `humorLevel` | `low` / `medium` / `high` | 幽默多少 |
| `periodStrictness` | `loose` / `normal` / `strict` | 年代细节严苛度（年代文更有用） |

### 叙事引擎可选对象 `narrative`（与上表并列）

存同一 JSON 根下的 **`narrative` 对象**，由后端与当前 `writingPipeline` 默认合并后注入章节提示词。字段均为可选（未写则用管线默认）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `emotionType` | 字符串 | 主情绪/基调标签，给作者与模型对齐用 |
| `intensityMin` / `intensityMax` | 数，0～1 | 情绪强度带宽 |
| `suppression` | 数，0～1 | 压抑度（越高越外冷） |
| `triggerFact` | 字符串 | 触发事实一句（不含评价） |
| `forbidden` | 字符串数组 | 禁止表述/套路，与默认禁止项**合并** |
| `rhythmHint` / `textureHint` / `povHint` | 字符串 | 节奏、语言材质、视角 intent |
| `affection` / `awkwardness` / `assertiveness` | 数，0～1 | 轻小说向微情绪维 |
| `interactionFocus` | 字符串 | 互动写法要求（如动作+对白+微反应） |
| `readerInferenceRule` | 布尔 | 为 true 时强调「删标签句仍能读懂情绪」 |

**M2 两阶段 Planner** 由运维在 **`application.yml`** 里开关（`novel.narrative-engine.two-phase-enabled`），不在本书 JSON 内配置；开启后章前多一次规划调用，失败自动退回单阶段。

**M3 叙事 Lint**：服务端在终稿后扫描禁止短语与（可选）标签句式；命中且开启 `lint-fix-enabled` 时会多一次窄幅修订，详见 **[API接口总表.md](./API接口总表.md)「§16.2.7」**。

创建示例：

```json
{
  "topic": "八十年代的供销社小事",
  "generationSetting": "偏写实",
  "hotMemeEnabled": false,
  "writingStyleParams": {
    "periodStrictness": "strict",
    "dialogueRatioHint": "medium",
    "humorLevel": "low"
  },
  "serializationPlatform": "未发布",
  "creatorNote": "短片合集素材库"
}
```

## 三、错误怎么展示给用户（不要暴露程序内部）

### 1）业务接口（`/api/novel/...` 里返回 `Map` 的）

成功：

- `status`: `"success"`
- `code`: `"OK"`
- `message`: 给人读的说明

失败：

- `status`: `"error"`
- `code`: 业务码，如 `TASK_CONFLICT`、`INVALID_STYLE_PARAMS`
- `message`: **只给用户看这一句**（已是中文短句，不再附带堆栈）

前端：`status !== 'success'` 时用 **`message` 做 Toast/弹窗**，不要用 `code` 当正文（`code` 可做埋点或映射按钮）。

### 2）返回 `ApiResponse` 的全局异常（如部分未单独 try/catch 的接口）

结构：`{ "code": 数字, "message": "...", "data": null }`

| HTTP | code | 典型 message |
|------|------|----------------|
| 400 | 400 | 参数类（可读中文） |
| 404 | 404 | 未找到该小说… |
| 409 | 409 | 状态冲突（可读中文） |
| 500 | 500 | 服务暂时不可用，请稍后重试 |

**禁止**把 `error.stack`、英文异常类名、SQL 片段展示给最终用户；需要排查问题时只看后端日志。

### 3）控制器里未预料异常

历史上会把 `e.getMessage()` 直接返回，容易泄露 JDBC/超时等英文。现已统一改为简短中文（`UserFriendlyExceptions`）。若仍看到生硬英文，多为 **IllegalStateException / IllegalArgumentException 里业务层写死的英文**，可再改业务文案为中文。

## 四、常见业务码（节选）

| code | 含义（给用户的话通常已在 message 里） |
|------|--------------------------------------|
| `TASK_CONFLICT` | 该章/区间正在生成，请稍后再试 |
| `INVALID_STYLE_PARAMS` | 文风微参格式不对或不包含有效字段 |
| `BOOK_META_UPDATE_FAILED` | 更新书籍备注失败，请重试 |
| `CREATE_TASK_FAILED` | 创建任务失败，请重试 |

---

## 五、与 AI 相关的字段别混淆

| 字段 | 作用 |
|------|------|
| `generationSetting` | 给模型的**写作设定**，参与生成 |
| `writingStyleParams` | 文风**微调**，参与生成 |
| `description` | **大纲/剧情规划**（模型产出） |
| `serializationPlatform` / `creatorNote` | **仅展示**，不参与生成 |
